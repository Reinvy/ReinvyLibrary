# Content Standards - ReinvyLibrary

This document establishes the official structure, formatting, and quality guidelines for all educational materials in **ReinvyLibrary**. These standards ensure that content is highly valuable for human developers and easily parsed/generated by **Agentic AIs**.

---

## 1. General Requirements

### 1.1 Bilingual Parity
Every topic must be available in both English and Indonesian:
- English base files must end with `.md` (e.g., `routing-basics.md`).
- Indonesian translations must end with `_id.md` (e.g., `routing-basics_id.md`).
- Filenames (kebab-case) must match exactly between the English and Indonesian versions.

### 1.2 Metadata Redundancy Ban
All metadata must be defined **only** in the YAML frontmatter at the top of the file.
- **Rule**: Do not add a `## Metadata` section or any similar table at the bottom of the document. This prevents duplicate data and formatting inconsistencies.

### 1.3 Strict Language Consistency
- English files (`locale: "en"`) must use English for all headings, text, and code comments.
- Indonesian files (`locale: "id"`) must use Indonesian for all headings, text, and code comments.
- Do not mix languages within headings (e.g., no `## Insight Penting` in an English file).

### 1.4 Code Quality
- All code blocks must use correct language syntax identifiers (e.g. `javascript`, `bash`, `prisma`).
- Code examples should be complete, realistic, and runnable rather than generic templates.
- Always include error handling in code examples.

## 2. Document Layout Standards

### 2.1 Tutorials (`type: "tutorial"`)
Tutorials provide step-by-step instructions. They must include the following H2 headings in the specified order:

#### English File Headings (Tutorials)
- `# Title of Tutorial` (H1)
- `## Summary`: A 2-3 sentence overview of the tutorial and final result.
- `## Target Audience`: Who this is for and expected developer level.
- `## Prerequisites`: Necessary prior knowledge, tools, and installations.
- `## Learning Objectives`: What the reader will learn (bullet points).
- `## Context and Motivation`: Why this topic is important and the real-world problem it solves.
- `## Core Content`: The step-by-step conceptual walkthrough.
- `## Code Examples`: Complete, runnable code blocks with explanations.
- `## Key Insights`: Best practices, gotchas, architecture notes, performance considerations.
- `## Next Steps`: Suggested topics to study next.
- `## Conclusion`: Summary of accomplishments.

#### Indonesian File Headings (Tutorials)
- `# Judul Tutorial` (H1)
- `## Ringkasan`: Ringkasan 2-3 kalimat mengenai tutorial dan hasil akhirnya.
- `## Target Audiens`: Siapa yang cocok mempelajari materi ini dan level pengembang.
- `## Prasyarat`: Pengetahuan, perkakas, dan instalasi yang diperlukan.
- `## Tujuan Pembelajaran`: Apa yang akan dipelajari pembaca (poin-poin).
- `## Konteks dan Motivasi`: Mengapa topik ini penting dan masalah nyata yang diselesaikan.
- `## Konten Inti`: Panduan konseptual langkah demi langkah.
- `## Contoh Kode`: Blok kode lengkap yang dapat dijalankan beserta penjelasannya.
- `## Insight Penting`: Praktik terbaik, jebakan, catatan arsitektur, pertimbangan performa.
- `## Langkah Berikutnya`: Topik yang disarankan untuk dipelajari selanjutnya.
- `## Kesimpulan`: Ringkasan pencapaian.

---

### 2.2 Syllabi (`type: "syllabus"`)
Syllabi outline structured courses. They must include:

#### English File Headings (Syllabi)
- `# Course Title` (H1)
- `## Overview`: Overall course scope and structure rationale.
- `## Curriculum`: Detailed week-by-week or module-by-module breakdown.
- `## Final Project`: Description of the practical project students will build.
- `## Assessment Criteria`: Evaluation breakdown (assignments, quizzes, project criteria).
- `## References`: Links to official documentation, videos, and books.

#### Indonesian File Headings (Syllabi)
- `# Judul Silabus` (H1)
- `## Ringkasan`: Lingkup keseluruhan kursus dan alasan struktur kurikulum.
- `## Kurikulum`: Rincian modul demi modul atau minggu demi minggu secara mendalam.
- `## Proyek Akhir`: Deskripsi proyek praktis yang akan dibangun siswa.
- `## Kriteria Penilaian`: Rincian evaluasi (tugas, kuis, kriteria proyek).
- `## Referensi`: Tautan ke dokumentasi resmi, video, dan buku referensi.

---

### 2.3 Cheat Sheets (`type: "cheatsheet"`)
Cheat Sheets provide quick command and syntax reference tables:

#### English File Headings (Cheat Sheets)
- `# Technology Cheat Sheet` (H1)
- `## Quick Reference Table`: A markdown table mapping Actions to Code and Descriptions.
- `## Common Commands`: Command-line interface references.
- `## Code Snippets`: Code syntax shortcuts.

#### Indonesian File Headings (Cheat Sheets)
- `# Cheat Sheet Teknologi` (H1)
- `## Tabel Referensi Cepat`: Tabel markdown yang memetakan Aksi ke Kode dan Deskripsi.
- `## Perintah Umum`: Referensi antarmuka baris perintah (CLI).
- `## Potongan Kode`: Pintasan sintaksis kode.

---

### 2.4 Guides (`type: "guide"`)
Guides introduce best practices or architectural designs:

#### English File Headings (Guides)
- `# Guide Title` (H1)
- `## Introduction`: Intro to the design pattern or guideline.
- `## Best Practices`: Highlighted rules with detailed rationale.
- `## Implementation Steps`: Architectural steps to apply the guide.

#### Indonesian File Headings (Guides)
- `# Judul Panduan` (H1)
- `## Pendahuluan`: Pengantar pola desain atau pedoman.
- `## Praktik Terbaik`: Aturan-aturan utama beserta alasan rasionalnya.
- `## Langkah Implementasi`: Langkah arsitektural untuk menerapkan panduan ini.
