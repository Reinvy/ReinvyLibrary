# Flutter App Development Syllabus

## Overview

This syllabus outlines a comprehensive path for mastering Flutter, Google's UI toolkit for building natively compiled applications. Students will start with Dart programming, move to basic and advanced UI development, master state management, and conclude by publishing functional cross-platform applications.

## Curriculum

### Module 1: Introduction & Dart Basics
- **Introduction to Flutter**: History, benefits, cross-platform architecture, and setting up Flutter SDK.
- **Dart Programming Fundamentals**: Variables, types, control flow, functions, object-oriented programming (OOP), asynchronous Dart (Futures, Stream, async/await).

### Module 2: Flutter Foundations & Layouts
- **Widgets System**: StatelessWidget vs StatefulWidget, BuildContext, widget tree.
- **Basic UI Widgets**: Container, Text, Row, Column, Image, ElevatedButton.
- **Layout & Scrolling**: Stack, Positioned, ListView, GridView, padding, and alignment.
- **Navigation**: Basic routing, named routes, passing data between pages.

### Module 3: State Management
- **Basic State Management**: setState, lifecycles of StatefulWidget.
- **Provider Pattern**: ChangeNotifierProvider, Consumer, Selector.
- **BLoC & Riverpod**: Core concepts, Event-to-State flows, and selecting appropriate managers for scalable apps.

### Module 4: API Integration & Data Persistence
- **Networking**: Making HTTP requests (http, dio), parsing JSON, and showing network indicators.
- **Local Storage**: Shared Preferences, SQLite, Hive.

### Module 5: Native Bridges & Performance Optimization
- **Platform Channels**: Communicating with Java/Kotlin (Android) and Objective-C/Swift (iOS).
- **Performance**: Profiling, debugging UI lag, memory leaks optimization.
- **Testing**: Unit testing, Widget testing, Integration testing.

## Final Project

Students will build a complete, production-ready cross-platform mobile app (e.g., e-commerce, weather app, or chat app) that:
- Connects to an external RESTful API.
- Implements persistent local storage.
- Uses Riverpod or BLoC for clean state management.
- Follows the Clean Architecture directory layout.

## Assessment Criteria

- **Assignments & Quizzes**: 30% (Dart fundamentals, layout exercises).
- **Intermediate Project**: 20% (Multi-page app UI, local data fetching).
- **Final Project & Presentation**: 50% (App functionality, performance, Clean Code, code review presentation).
